EXP 01
Install Linux (Ubuntu) using VirtualBox
Aim: Install Ubuntu OS as a virtual machine and study Linux features
💻
How to install Ubuntu in VirtualBox
What the exam will ask you: explain the installation steps or architecture
1
DOWNLOAD Get the Ubuntu ISO file
Go to ubuntu.com/download/desktop → download the .iso file. This is like a CD image of the OS installer. Save it somewhere you'll find it.
2
VIRTUALBOX Create a new virtual machine
Open VirtualBox → Click New → Fill in:
• Name: Ubuntu  |  Type: Linux  |  Version: Ubuntu (64-bit)
• RAM: give it half your physical RAM (e.g., 2GB if you have 4GB)
• Hard disk: Create VDI → Dynamically allocated → minimum 25GB
3
ATTACH ISO Put the ISO into the virtual CD drive
Settings → Storage → Click "Empty" under Controller: IDE → Click the disc icon on the right → Choose your .iso file
4
START Boot and install Ubuntu
Click Start → The VM boots from the ISO → Choose Install Ubuntu → Follow on-screen prompts → When asked about disk, choose "Erase disk and install Ubuntu" (don't worry — it erases the VIRTUAL disk, not your real laptop!)
5
RESULT Ubuntu is now installed inside VirtualBox
After install, remove the ISO (Storage settings → remove disc) and restart the VM. You'll see Ubuntu's desktop. Open Terminal with Ctrl+Alt+T
Linux Architecture — Draw This if Asked
Your App (vi, browser, script) ↓ talks to Shell (bash — reads your commands) ↓ passes to Kernel (core OS — manages CPU, RAM, disk) ↓ controls Hardware (your actual computer)
BASICWhat is VirtualBox? What is the difference between host OS and guest OS?›

VirtualBox is a free program that lets you run a whole operating system inside a window on your computer — like a computer inside a computer.

Host OS = the OS on your actual laptop (e.g., Windows or macOS). It runs VirtualBox.
Guest OS = the OS running inside VirtualBox (Ubuntu in our case).

⚡ Why use a VM? Safe to experiment. If you break Ubuntu, just delete it and reinstall. Your Windows/Mac is untouched.
BASICName 4 important features of the Linux operating system.›
FeatureWhat it means
Multi-userMany users can use the same Linux machine at the same time
Multi-taskingMany programs can run at the same time
Open SourceSource code is free — anyone can read, modify, share it
SecureStrong permission system — users can only access their own files
PortableRuns on almost any hardware — phones (Android), servers, supercomputers
INTERMEDIATEExplain the architecture of Linux. What does the Kernel do?›

Linux has 4 layers stacked on top of each other:

1. Hardware — Physical computer: CPU, RAM, hard disk.
2. Kernel — The brain of the OS. Directly controls hardware. Manages processes, memory, files, devices. You never interact with the kernel directly.
3. Shell — The translator. Takes commands you type (like ls) and tells the kernel what to do. Bash is the most common shell.
4. User Applications — Programs you actually use: text editor, browser, your shell scripts.

BASICWhat is the man command? How do you exit it?›

man ls opens the full manual (help page) for the ls command. It shows every option, what each flag does, and examples.

Press q to quit. Press / then type a word to search within the manual. Press n for next match.

BASICWhat does echo do? What is the difference between echo $USER and echo '$USER'?›

echo just prints text to the screen. It's the Linux version of print().

echo "Hello"        # prints: Hello
echo $USER          # prints your username (expands the variable)
echo '$USER'        # prints: $USER (single quotes = taken literally)

Double quotes allow variables to expand. Single quotes treat everything as plain text.

id — shows your UID (user ID number), GID (group ID), and which groups you belong to.

which python3 — shows the full path to where a command is stored on disk. Useful when you have multiple versions installed.

EXP 2B
Vi / Vim Editor
Aim: Create, edit, and save files using the vi text editor
📝
How to use Vi in the practical
Vi has 3 modes — understanding this is everything
1
OPEN Open a file in vi
vi myfile.txt
You are now in NORMAL mode (default). The cursor blinks but nothing you type appears as text yet. Keys are commands here, not letters.
2
TYPE Start typing text → press i first
i
Pressing i switches to INSERT mode. You'll see -- INSERT -- at the bottom. Now type whatever you want. This is the only mode where your text actually appears.
3
IMPORTANT Go back to Normal mode when done typing
Esc
Press the Esc key. The -- INSERT -- at the bottom disappears. You are back in Normal mode. Press Esc whenever you are confused or stuck.
4
SAVE + QUIT Save your file and exit
:wq
Make sure you're in Normal mode (press Esc first), then type :wq and press Enter. :w = write (save), :q = quit. This is what you'll use 95% of the time.
5
EXIT WITHOUT SAVING Made a mistake? Exit without saving
:q!
The ! means "force". This quits WITHOUT saving any changes. Your file goes back to how it was before.
Quick Reference — Essential Vi Keys (all in Normal mode after pressing Esc)
i→ INSERT mode (type text before cursor)
a→ INSERT mode (type text AFTER cursor)
o→ INSERT mode on a NEW line below
ddDelete entire current line
yy then pCopy (yank) current line, then Paste below
uUndo last action
:5Jump to line number 5
/word + EnterSearch for "word" (press n for next match)
:%s/old/new/gReplace ALL "old" with "new" in the file
⚠ Most common mistake: you start typing but letters are doing strange things (deleting, moving cursor). This means you're in NORMAL mode. Just press Esc then i to get back to INSERT mode.
BASICWhat are the 3 modes of the vi editor? How do you switch between them?›
ModeHow to enterWhat you can do
NormalDefault / Press EscNavigate, delete, copy — keyboard keys are commands
InsertPress i (or a, o)Type and edit text like a normal editor
Command (Last Line)Press : from Normal:w save, :q quit, :wq save+quit, :q! force quit
⚡ If you are lost in vi: press Esc multiple times until you're in Normal mode, then type :wq to save and exit safely.
INTERMEDIATEHow do you replace every occurrence of the word "linux" with "LINUX" in vi?›
:%s/linux/LINUX/g

Breakdown: : = enter command mode | % = all lines | s = substitute | /linux/ = find this | /LINUX/ = replace with this | /g = global (all occurrences on each line)

Without % it only works on current line. Without g it only replaces the first match per line.

EXP 3B
File System Management Commands
Aim: Create, copy, move, delete files and directories; use grep, sort, wc, pipe
📁
File management — type these in the terminal
Each command is followed by what you'll see on screen
1
CREATE Make a new directory and go into it
mkdir myproject cd myproject pwd
/home/student/myproject
mkdir = make directory. cd = change directory (move into it). pwd confirms where you are now.
2
CREATE FILE Create a file with some text
echo "mango apple banana mango" > fruits.txt cat fruits.txt
mango apple banana mango
> redirects the output into a file (creates it). cat shows the file's contents.
3
LIST See files in the folder
ls ls -l
fruits.txt -rw-r--r-- 1 student student 25 Apr20 fruits.txt
ls = list. ls -l = long format showing permissions, size, date. ls -la also shows hidden files.
4
COPY / MOVE Copy and rename a file
cp fruits.txt fruits_backup.txt mv fruits_backup.txt foods.txt ls
foods.txt fruits.txt
cp copies. mv moves OR renames (here it renamed from fruits_backup.txt to foods.txt).
5
COUNT Count lines, words, characters
wc fruits.txt
1 4 25 fruits.txt
Output = lines | words | bytes | filename. Use wc -l for lines only, wc -w for words only, wc -c for characters.
6
SEARCH + PIPE Search for a word using grep and pipe
grep "mango" fruits.txt cat fruits.txt | grep "mango"
mango apple banana mango
Both commands do the same thing. The | (pipe) sends the output of cat as input to grep. Pipes let you chain commands.
7
SORT + TRANSFORM Sort and convert case
sort fruits.txt cat fruits.txt | tr 'a-z' 'A-Z'
apple banana mango mango MANGO APPLE BANANA MANGO
sort arranges lines alphabetically. tr 'a-z' 'A-Z' translates (converts) lowercase to uppercase.
8
DANGER Delete a file (permanent!)
rm foods.txt
No Recycle Bin in Linux. Once deleted, it's gone forever. Use rm -r foldername to delete a folder and everything inside it.
BASICWhat do the letters like rw-r--r-- mean in ls -l output?›

It shows who can do what with the file. There are 3 groups of 3 letters each:

-  rw-  r--  r--
↑   ↑    ↑    ↑
|  Owner Group Others
File type (- = file, d = directory)

Each group: r = read | w = write | x = execute | - = not allowed

So rw-r--r-- means: Owner can read+write. Group can only read. Others can only read.

BASICWhat is a pipe |? Give a real example.›

A pipe takes the output of one command and feeds it as input to the next command. Like a water pipe — output flows through to the next stage.

ls -l | grep ".txt"      # list files, then keep only .txt ones
cat file.txt | wc -l     # show file, then count the lines
cat file.txt | sort | uniq  # show → sort → remove duplicates
INTERMEDIATEWhat does chmod 755 script.sh do?›

chmod changes file permissions using numbers. r=4, w=2, x=1. Add them up for each group.

chmod 755 script.sh
7 = 4+2+1 = rwx  → Owner: read, write, execute
5 = 4+0+1 = r-x  → Group: read, execute
5 = 4+0+1 = r-x  → Others: read, execute

chmod +x script.sh  # simpler: just add execute for everyone
⚡ You MUST run chmod +x script.sh before running any shell script. Otherwise you get "Permission denied".
EXP 3C
Compression & Archiving — tar, gzip, zip
Aim: Compress and archive files using tar, gzip, bzip2, zip
🗜️
Compress and extract files — step by step
First create some test files, then practice archiving them
1
SETUP Create some test files to work with
mkdir testfolder echo "File one" > testfolder/a.txt echo "File two" > testfolder/b.txt echo "File three" > testfolder/c.txt
Now you have a folder with 3 files to practice archiving.
2
ZIP Zip the folder (Windows-compatible)
zip -r myarchive.zip testfolder/ ls -lh myarchive.zip
-r means recursive (include all files inside the folder). ls -lh shows the file size in human-readable format so you can see it got smaller.
3
UNZIP Extract a zip file
unzip myarchive.zip -d extracted/ ls extracted/
-d specifies the destination folder. Without -d it extracts into current directory.
4
GZIP Compress a single file with gzip
cp testfolder/a.txt . gzip a.txt ls
a.txt.gz (a.txt is GONE — replaced by a.txt.gz)
Important: gzip deletes the original and creates a .gz file. To get it back: gunzip a.txt.gz
5
TAR Create a .tar.gz archive (most common in Linux)
tar -cvzf backup.tar.gz testfolder/
testfolder/ testfolder/a.txt testfolder/b.txt testfolder/c.txt
Flags: c=create, v=verbose (show files), z=compress with gzip, f=filename. The output shows each file being added.
6
TAR EXTRACT Extract a .tar.gz archive
tar -xvzf backup.tar.gz
Change c to x — that's the only difference between create and extract. To list contents without extracting: tar -tvzf backup.tar.gz
Memory trick for tar flags
create verbose zip(gzip) filename ← Create .tar.gz extract verbose zip(gzip) filename ← Extract .tar.gz
BASICWhat is the difference between archiving and compression? Why do we use both together?›

Archiving = bundling many files into ONE file (like putting them in a bag). No size reduction. tar does this.
Compression = making the file smaller using math. gzip, bzip2, zip do this.

We use both together with tar -czf archive.tar.gz folder/ because tar bundles everything first, then gzip shrinks the bundle.

INTERMEDIATEWhat happens to the original file when you run gzip file.txt?›

The original file is deleted. Only the compressed file.txt.gz remains. This is different from zip which keeps the original.

gzip file.txt        # file.txt → DELETED, file.txt.gz created
gzip -k file.txt     # KEEP original (both exist after)
gunzip file.txt.gz   # .gz deleted, original file.txt restored

User Management Commands
Aim: Add, delete, and manage users and groups in Linux
👤
Managing users — step by step
Most commands need sudo (admin privileges)
1
CHECK See who is logged in
whoami who w
whoami = just your name. who = all logged-in users. w = all users + what they are doing.
2
ADD USER Create a new user called "john"
sudo useradd -m john sudo passwd john
-m creates a home folder at /home/john. After sudo passwd john, you'll be asked to type a new password for john twice.
3
SWITCH USER Log in as john
su john whoami exit
john
su = switch user. Type john's password. whoami now shows "john". exit goes back to your original user.
4
GROUPS Create a group and add john to it
sudo groupadd developers sudo usermod -aG developers john groups john
john : john developers
-aG = append to Group (don't remove from existing groups). groups john confirms john is now in the developers group.
5
OWNERSHIP Change who owns a file
touch report.txt sudo chown john report.txt ls -l report.txt
-rw-r--r-- 1 john student 0 Apr20 report.txt
The owner changed from "student" to "john". Now only john (and root) can modify it.
6
DELETE USER Remove the user john
sudo userdel -r john
-r = remove home directory too. Without -r, the account is deleted but /home/john remains.
Key files — examiner often asks "where is this stored?"
cat /etc/passwdAll users listed here (username:x:UID:GID:name:home:shell)
cat /etc/groupAll groups and their members
cat /etc/shadowEncrypted passwords (only root can read)
BASICWhat is the difference between sudo and su?›
CommandWhat it doesPassword
sudo cmdRun ONE command as admin (root), then back to normalYour own password
su johnCompletely switch to user john's shell sessionJohn's password
sudo suBecome the root user permanently in this sessionYour own password

Think of sudo like "run this one task as admin". Think of su like "actually become that user".

INTERMEDIATEExplain the format of the /etc/passwd file. What does each field mean?›

PartMeaning

EXP 05
Process Management Commands
Aim: View, control, and kill running processes; use fg, bg, nice
⚙️
Process management — step by step
A "process" is just a running program. Every command you run creates a process.
1
VIEW See all running processes
ps ps aux
USER PID %CPU %MEM ... COMMAND student 1234 0.0 0.1 ... bash
ps shows just your current processes. ps aux shows ALL processes from ALL users. The PID (Process ID) is the unique number for each process.
2
LIVE VIEW Watch processes updating live
top
Shows real-time CPU and memory usage. Press q to quit. Shows which processes are using the most resources at the top. Press k then type a PID to kill a process from inside top.
3
BACKGROUND Run a process in the background
sleep 60 & jobs
[1] 5678 [1]+ Running sleep 60 &
The & at the end sends the process to the background. Your terminal is free to use. jobs shows background tasks. The [1] is the job number.
4
SUSPEND + BG Pause a foreground process then resume it in background
sleep 100 (now press Ctrl+Z) bg
[1]+ Stopped sleep 100 [1]+ sleep 100 &
Ctrl+Z suspends (pauses) the process. Then bg sends it to background to continue running. fg brings it back to foreground.
5
KILL Stop a running process
ps aux | grep sleep kill 5678
Find the PID with ps aux | grep processname, then kill PID. If it doesn't stop: kill -9 PID (force kill — cannot be ignored).
6
NICE Start a process with lower priority
nice -n 10 sleep 100 & ps aux | grep sleep
Priority range: -20 (most CPU) to +19 (least CPU). Setting +10 means this process yields to others. Good for long background tasks like backups.
BASICWhat is a process? What is a PID?›

A process is a program that is currently running. Every time you type a command, Linux creates a process to run it. Multiple copies of the same program can run at the same time — each is a separate process.

PID = Process ID. Every process gets a unique number. Use this number to identify and control the process (kill it, change its priority, etc.).

INTERMEDIATEWhat is the difference between kill 1234 and kill -9 1234?›

kill 1234 sends signal 15 (SIGTERM) — politely asks the process to stop. The process can save its work and shut down cleanly. It can even ignore this signal.

kill -9 1234 sends signal 9 (SIGKILL) — the kernel immediately destroys the process. No cleanup, no saving. The process cannot ignore or block this.

⚠ Always try regular kill first. Only use -9 if the process is frozen and not responding.
EXP 06
Memory Management Commands
Aim: Check RAM usage, disk space, and virtual memory statistics
🧠
Check memory and disk — step by step
Run each command and understand what each line of output means
1
RAM Check how much memory is in use
free -h
total used free shared buff/cache available Mem: 7.7G 2.1G 1.2G 0.3G 4.4G 5.1G Swap: 2.0G 0.0G 2.0G
Mem row = physical RAM. Swap row = virtual RAM (disk used as extra RAM). The available column tells you how much free RAM your programs can actually use right now.
2
DISK SPACE See how full each hard disk partition is
df -h
Filesystem Size Used Avail Use% Mounted on /dev/sda1 50G 12G 35G 26% /
df = Disk Filesystem. The Use% column is the key one — if it hits 95%+ the system has problems. -h shows GB/MB instead of confusing byte counts.
3
FOLDER SIZE Find out how big a specific folder is
du -sh /home/student/ du -sh *
45M /home/student/
du = Disk Usage. -s = summary (one total, not every file). -h = human readable. * shows size of each item in current folder.
4
DETAILED RAM See detailed memory info from the kernel
cat /proc/meminfo | head -10
MemTotal: 8058944 kB MemFree: 1234567 kB MemAvailable: 5678901 kB ...
/proc/meminfo is a special virtual file the kernel creates to show live memory data. head -10 shows just the first 10 lines (it's a long file).
5
VMSTAT See CPU, memory, and I/O statistics together
vmstat
procs memory swap io system cpu r b swpd free buff cache si so bi bo in cs us sy id wa
vmstat gives a one-line snapshot of everything. Key columns: free=free RAM, us=user CPU%, sy=system CPU%, id=idle CPU%.
BASICWhat is swap memory? Why is it needed?›

Swap is a portion of the hard disk that Linux uses as extra RAM when physical RAM runs out. When RAM is full, the OS moves less-used data to the swap space to make room for new data.

The problem: hard disks are 100x slower than RAM. So when swap is being used heavily, the system becomes very slow (you'll hear the disk clicking and everything freezes).

Think of RAM as your desk and swap as a drawer — when the desk is full you put things in the drawer, but getting them back takes longer.

BASICWhat is the difference between df and du?›

df (Disk Filesystem) shows how full each partition/drive is overall — like checking the total capacity of your hard drive.

du (Disk Usage) shows how much space specific files and folders are taking — like checking how much space one folder is using.

⚡ Practical use: df -h tells you IF disk is almost full. Then du -sh * | sort -h tells you WHICH folder is eating all the space.
EXP 07
Basic Shell Scripting
Aim: Write scripts for arithmetic, simple interest, and largest of 3 numbers
🔧
Write and run your first shell scripts
Every script follows the same 3 steps: Write it → Make it executable → Run it
1
ALWAYS DO FIRST The 3 steps to create any script
Step A: Open vi and write the script vi myscript.sh Step B: Make it executable chmod +x myscript.sh Step C: Run it ./myscript.sh
You need chmod +x only once per file. After that, ./myscript.sh runs it every time.
2
SCRIPT 1 Arithmetic operations
#!/bin/bash # Every script starts with this line read -p "Enter two numbers: " a b echo "Addition : $((a + b))" echo "Subtraction : $((a - b))" echo "Multiplication: $((a * b))" echo "Division : $((a / b))" echo "Remainder : $((a % b))"
The shebang #!/bin/bash MUST be the first line. $((...)) is how bash does math. read -p "text" var shows a prompt and stores input in var.
3
SCRIPT 2 Simple Interest calculator
#!/bin/bash read -p "Enter Principal: " p read -p "Enter Rate (%): " r read -p "Enter Time (years): " t si=$(( (p * r * t) / 100 )) echo "Simple Interest = $si"
Formula: SI = (P × R × T) / 100. Variables have no spaces around = sign. Access with $ before the name.
4
SCRIPT 3 Largest of 3 numbers
#!/bin/bash read -p "Enter 3 numbers: " a b c if [ $a -gt $b ] && [ $a -gt $c ]; then echo "Largest: $a" elif [ $b -gt $c ]; then echo "Largest: $b" else echo "Largest: $c" fi
Comparison operators: -gt=greater than, -lt=less than, -eq=equal, -ne=not equal. Conditions go inside [ ] with spaces.
Common mistakes beginners make
❌ x = 5 WRONG — spaces around = cause an error. Correct: x=5
❌ echo x + y WRONG — won't add. Correct: echo $((x + y))
❌ if [$a -gt $b] WRONG — must have spaces inside brackets. Correct: if [ $a -gt $b ]
BASICWhat is the shebang line #!/bin/bash? Why must it be the first line?›

#! (called "shebang" or "hashbang") tells the OS which program to use to run this script. /bin/bash is the path to the bash shell.

When you run ./script.sh, the OS reads the first two bytes. If it sees #!, it sends the rest of the script to the specified interpreter. Without this line, the OS might use the wrong shell and your script could fail.

INTERMEDIATEWhat are the comparison operators for numbers in bash? How do they differ from math symbols?›
Math symbolBash operatorMeaning
>-gtgreater than
<-ltless than
>=-gegreater than or equal
<=-leless than or equal
==-eqequal to
!=-nenot equal to

For string comparison use = and != (without dash). E.g., if [ "$name" = "Alice" ]

EXP 08
Advanced Shell Scripting
Aim: case statement calculator, star pattern, file/directory existence check
⚡
Advanced scripts — 3 programs to know
case statement, loops for patterns, and file testing
1
SCRIPT 1 Menu calculator using case statement
#!/bin/bash echo "1) Add 2) Subtract 3) Multiply 4) Divide" read -p "Choose (1-4): " choice read -p "Enter two numbers: " a b case $choice in 1) echo "Result: $((a + b))" ;; 2) echo "Result: $((a - b))" ;; 3) echo "Result: $((a * b))" ;; 4) echo "Result: $((a / b))" ;; *) echo "Invalid choice!" ;; esac
case $variable in ... esac. Each option ends with ;; (two semicolons). *) is the default case (like else). esac = case backwards = end of case.
2
SCRIPT 2 Print star triangle pattern
#!/bin/bash read -p "How many rows? " n for ((i=1; i<=n; i++)); do for ((j=1; j<=i; j++)); do echo -n "* " done echo # this adds a newline after each row done
* * * * * * * * * *
Outer loop: controls which row you're on. Inner loop: controls how many stars on this row. echo -n prints without newline so stars stay on same line.
3
SCRIPT 3 Check if file/directory exists, create if not
#!/bin/bash read -p "Enter name to check: " name if [ -f "$name" ]; then echo "'$name' exists and is a FILE" elif [ -d "$name" ]; then echo "'$name' exists and is a DIRECTORY" else echo "'$name' does not exist. Creating it..." mkdir "$name" echo "Directory '$name' created!" fi
-f checks for a FILE. -d checks for a DIRECTORY. These are "test flags". Always quote your variable "$name" in case it has spaces.
All File Test Flags (used in if conditions)
-f "$path"True if it's a regular FILE
-d "$path"True if it's a DIRECTORY
-e "$path"True if it EXISTS (file or directory)
-r "$path"True if it's READABLE
-x "$path"True if it's EXECUTABLE
-s "$path"True if it exists and is NOT empty
BASICWhat is a case statement? When should you use it instead of if-else?›

A case statement matches one variable against many possible values. Use it when you have 3+ options based on the same variable — it's much cleaner than stacking if-elif-elif-elif.

case $option in
  "yes"|"y") echo "You said yes" ;;   # | means OR
  "no"|"n")  echo "You said no" ;;
  [0-9])     echo "You typed a digit" ;;   # pattern!
  *)         echo "Unknown input" ;;    # default case
esac
INTERMEDIATETrace through the star pattern script for n=3. Why are there two loops?›

The outer loop (i) controls the row number. The inner loop (j) controls how many stars to print on that row.

i (row)j runs 1 to iWhat prints
1j=1*
2j=1,2* *
3j=1,2,3* * *

echo -n "* " doesn't add a newline, so stars stay on the same row. The plain echo at the end of the outer loop adds the newline to finish each row.

EXP 09
grep, egrep & sed Commands
Aim: Search files with grep/egrep; find+replace and duplicate lines with sed
🔍
grep and sed — the most powerful text tools
First create a test file, then run each command on it
1
SETUP Create a test file to work with
cat > sample.txt << EOF Hello World This is Linux Linux is powerful hello again Script file: test.sh Image file: photo.png EOF
This creates sample.txt with 6 lines. EOF means "end of file" — type it to finish. Now you have something to search in.
2
GREP Find lines containing a word
grep "Linux" sample.txt

Grep prints every line that contains "Linux". Case sensitive — it won't show "linux" (lowercase).
3
GREP FLAGS Count lines, show line numbers, ignore case
grep -c "Linux" sample.txt # count matches grep -n "Linux" sample.txt # show line numbers grep -i "hello" sample.txt # case insensitive grep -v "Linux" sample.txt # lines WITHOUT Linux
2 2: This is Linux 3: Linux is powerful Hello World hello again Script file: test.sh Image file: photo.png
These flags are very commonly asked. Remember: -c=count, -n=number, -i=ignore case, -v=invert.

ls /bin/ | egrep "\.sh$" egrep "\.sh$|\.png$" sample.txt
Script file: test.sh
egrep = extended grep. The | means OR (show .sh OR .png). The $ means "ends with". The \. is a literal dot (. alone means "any character" in regex).

Output is: lines words bytes filename. The experiment asks you to count all three using grep/wc in a script.

s/old/new/g = substitute old with new, globally (all occurrences). Without -i it just prints the result — the actual file is unchanged. With -i it modifies the file in place.

-n = suppress auto-printing. The first p prints the line once, the second p prints it again. Result: every line appears twice.
BASICWhat does grep stand for and what does it do?›

grep = Global Regular Expression Print. It searches through file(s) line by line and prints every line that matches your search pattern.

grep "error" log.txt         # find all lines with "error"
grep -i "error" log.txt      # same but case-insensitive
grep -c "error" log.txt      # just show the COUNT of matches
grep -n "error" log.txt      # show line numbers too
grep -v "error" log.txt      # show lines WITHOUT "error"
INTERMEDIATEWhat is sed? What is the difference between sed 's/old/new/g' and sed -i 's/old/new/g'?›

sed = Stream Editor. It processes text line by line and applies changes. Without -i, it only shows the result on screen — the actual file is NOT changed. With -i, it edits the actual file.

sed 's/cat/dog/g' pets.txt       # shows changed output on screen
# pets.txt still has "cat" in it!

sed -i 's/cat/dog/g' pets.txt    # actually changes pets.txt
# pets.txt now has "dog" everywhere

AWK & Perl Scripting
Aim: AWK for even numbers & Fibonacci; Perl for sorting arrays and prime check
🦅
AWK and Perl — the 4 programs you need to know
Write each in vi, chmod +x, then run with ./filename
1
AWK PROG 1 Print even numbers in a range
# Save as even.sh, then run it #!/bin/bash read -p "Start: " s read -p "End: " e awk -v start=$s -v end=$e 'BEGIN { for (i = start; i <= end; i++) { if (i % 2 == 0) print i } }'
2 4 6 8 10 (if you entered 1 to 10)
-v start=$s passes shell variable $s into AWK as variable start. AWK runs the code inside BEGIN{ } once before reading any input. % is modulus (remainder after division) — even numbers have remainder 0 when divided by 2.
2
AWK PROG 2 Fibonacci series
#!/bin/bash read -p "How many terms? " n awk -v n=$n 'BEGIN { a = 0 b = 1 for (i = 1; i <= n; i++) { print a temp = a + b a = b b = temp } }'
0 1 1 2 3 5 8 13 ... (for n=8)
Fibonacci: each number = sum of previous two. Start with a=0, b=1. Each step: print a, then calculate next term by swapping. The temp variable is needed to avoid losing the old value of a.
3
PERL PROG 1 Sort elements of an array
# Save as sort.pl #!/usr/bin/perl use strict; use warnings; my @nums = (15, 3, 9, 1, 22, 7); print "Original: @nums\n"; my @sorted = sort { $a <=> $b } @nums; print "Sorted: @sorted\n";
Original: 15 3 9 1 22 7 Sorted: 1 3 7 9 15 22
Run with: perl sort.pl. The <=> (spaceship) operator does numeric comparison. Without it, 10 would come before 2 (alphabetical order). Arrays in Perl start with @, scalar variables with $.
4
PERL PROG 2 Check if a number is prime
# Save as prime.pl #!/usr/bin/perl print "Enter a number: "; my $n = <STDIN>; chomp $n; # remove the Enter key (\n) from input my $prime = 1; if ($n < 2) { $prime = 0; } else { for (my $i = 2; $i * $i <= $n; $i++) { if ($n % $i == 0) { $prime = 0; last; # stop checking, it's not prime } } } if ($prime) { print "$n is Prime\n"; } else { print "$n is Not Prime\n"; }
<STDIN> reads user input. chomp removes the trailing newline that comes with keyboard input. Loop only goes up to √n for efficiency. last breaks out of the loop immediately.
BASICWhat is AWK? What are $1, NF, and NR?›

AWK is a text processing tool. It reads each line and splits it into fields (columns) separated by spaces. It's named after its authors: Aho, Weinberger, Kernighan.

VariableWhat it holdsExample (for line "John 25 Mumbai")
$0Entire current lineJohn 25 Mumbai
$1First fieldJohn
$2Second field25
$3Third fieldMumbai
NFNumber of Fields on this line3
NRNumber of current Row (line number)1 for first line, 2 for second...
INTERMEDIATEWhy do we use chomp in Perl after reading input?›

When the user types something and presses Enter, Perl gets the text plus the Enter key (which is the newline character \n). So if user types 5, Perl stores "5\n".

chomp removes that trailing newline. Without it, comparisons and printing can give weird results (extra blank lines, comparisons failing).

my $n = <STDIN>;    # user types 5 → $n = "5\n"
chomp $n;           # now $n = "5"  (clean)

Symbolic Links & Hard Links
Aim: Create symlinks and hardlinks, then observe what happens when original is deleted
🔗
Create links and test deleting original
This is a demo experiment — just run these commands in order
1
SETUP Create an original file
echo "This is the original content" > original.txt cat original.txt
This is the original content
We'll create both a symlink and a hardlink pointing to this file, then delete it and see what happens.
2
SYMLINK Create a symbolic (soft) link
ln -s original.txt symlink.txt ls -l
-rw-r--r-- 1 student student 29 ... original.txt lrwxrwxrwx 1 student student 12 ... symlink.txt -> original.txt
Notice: symlink shows l at the start of permissions and shows -> original.txt. It's just a pointer to the original file's location.
3
HARDLINK Create a hard link
ln original.txt hardlink.txt ls -li
12345 -rw-r--r-- 2 student ... original.txt 12345 -rw-r--r-- 2 student ... hardlink.txt
The -i flag shows inode numbers. Both original.txt and hardlink.txt have the same inode number (12345) — they point to the exact same data on disk. The "2" means 2 hard links exist.
4
THE KEY TEST Delete the original and see what happens
rm original.txt cat symlink.txt # test symlink after deletion cat hardlink.txt # test hardlink after deletion
cat: symlink.txt: No such file or directory ← BROKEN! This is the original content ← STILL WORKS!
This is the most important result of the experiment. Symlink breaks because its target file is gone. Hardlink still works because it references the actual data directly — the data only gets deleted when ALL hardlinks to it are gone.
Symlink vs Hardlink — Summary (examiner WILL ask this)
Soft/Symbolic Link Hard Link Created with: ln -s source link ln source link What it is: Pointer to file path Direct reference to data Inode: Different inode SAME inode as original If original Link is BROKEN Data still accessible is deleted: (dangling symlink) (survives!) Can cross disk: YES NO (same disk only)
BASICWhat is a symbolic link? Give a simple real-world analogy.›

A symbolic link is like a shortcut on Windows Desktop. It's a small file that just contains the path to the real file. When you open the shortcut, the OS follows the path and opens the real file.

Key behaviour: if you delete the original file, the shortcut still exists but it leads nowhere (broken shortcut).

ln -s /home/alice/documents/report.pdf my_report.pdf
# my_report.pdf is a shortcut to the actual report
# They show the same content
# Delete report.pdf → my_report.pdf is broken
INTERMEDIATEWhat is an inode? Why do hardlinks share the same inode?›

An inode is a record on disk that stores a file's information: size, permissions, owner, timestamps, and location of actual data on disk. It does NOT store the filename.

A hard link creates a new directory entry pointing to the same inode. Both filenames refer to the same data. When you delete a filename, you reduce the inode's "link count" by 1. The data is actually deleted only when link count reaches 0 (no more names pointing to it).

⚡ Think of an inode as a house (the data). Hard links are different addresses for the same house. Destroying one address doesn't demolish the house — you need to destroy ALL addresses first.

Task Scheduling — crontab
Aim: Schedule a script to run automatically using crontab; automate system health monitoring
⏰
Schedule a task to run automatically
crontab lets you set "run this script every 5 minutes" — the OS does it automatically
1
UNDERSTAND The 5-field cron time syntax
Position: MIN HOUR DAY MONTH WEEKDAY Range: 0-59 0-23 1-31 1-12 0-7 * means "every value" (every minute, every hour, etc.) Examples: * * * * * # every single minute 0 * * * * # every hour at minute 0 0 9 * * * # every day at 9:00 AM */5 20 * * * # every 5 min during 8 PM hour 0 0 1 * * # midnight on 1st of each month
The format from the experiment: */5 20 * * * means every 5 minutes, during hour 20 (8 PM), every day, every month.
2
WRITE SCRIPT Create the script to be scheduled
vi /home/student/create_file.sh
#!/bin/bash # This script creates a file with current timestamp touch /home/student/Desktop/crontest_$(date +%s).txt echo "Script ran at $(date)" >> /home/student/cronlog.txt
After writing, press Esc then :wq to save. The $(date +%s) creates a unique filename using Unix timestamp so each run creates a new file.
3
PERMISSION Make the script executable
chmod +x /home/student/create_file.sh
Always required. If you forget this step, cron will say "permission denied" and the job won't run.
4
ADD JOB Edit crontab to schedule the script
crontab -e
This opens a text editor (usually nano or vi). Add this line at the bottom:
*/5 20 * * * /home/student/create_file.sh
Save and exit. Cron will now run it every 5 minutes during hour 20.
5
VERIFY Check your scheduled jobs
crontab -l
*/5 20 * * * /home/student/create_file.sh
Shows all your scheduled cron jobs. Wait 5 minutes, then check your Desktop — you should see new files appearing. Also check cat /home/student/cronlog.txt to see the log.
6
BONUS Health monitoring script (system monitoring task)
#!/bin/bash LOG="/home/student/health.log" echo "=== $(date) ===" >> $LOG echo "CPU idle: $(top -bn1|grep 'Cpu'|awk '{print $8}')%" >> $LOG echo "RAM free: $(free -m|awk 'NR==2{print $4}') MB" >> $LOG echo "Disk used: $(df -h /|awk 'NR==2{print $5}')" >> $LOG
Add to crontab as: */10 * * * * /home/student/health.sh to run every 10 minutes and log system stats.
⚠ Most common reason cron doesn't work: You used a relative path like ./script.sh in crontab. Cron doesn't know your current directory. ALWAYS use the full path like /home/student/script.sh
BASICWhat is cron? What is crontab?›

cron is a background service (daemon) running all the time that executes scheduled tasks at specific times. It's like an alarm clock built into Linux.

crontab is your personal schedule file where you write what to run and when. Each user has their own crontab. The cron daemon reads everyone's crontab files and runs the jobs at the right time.

Real-world use: automatic database backups at midnight, sending reports every Monday morning, clearing temp files every week.

INTERMEDIATEYour cron job isn't running even though the crontab entry looks right. What should you check?›

Most common reasons cron jobs fail silently:

1. Relative paths — cron has no working directory. Use /home/user/script.sh not ./script.sh.

2. No execute permission — did you run chmod +x script.sh? Test with ls -l script.sh — should show x in permissions.

3. Script has errors — test it manually first: bash /home/user/script.sh. If it doesn't work manually, it won't work in cron.

4. Check cron logs: grep CRON /var/log/syslog
